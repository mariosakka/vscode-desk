Examples & Patterns

Real workflows showing how humans and AI agents use Relay together.

1. Agent builds a prototype doc agent

An agent is asked to document a new API endpoint. It creates a structured .relay page with a custom callout style, then adds a bookmark linking to it.

Step 1 — create the page

{
  "name": "create_page",
  "arguments": {
    "filename": "api-auth.relay",
    "title": "Auth API",
    "content": "<h2>POST /auth/login</h2>
<div class=\"callout tip\">
  <strong>Base URL:</strong> https://api.example.com
</div>
<h3>Request body</h3>
<pre><code>{ \"email\": string, \"password\": string }</code></pre>
<h3>Response</h3>
<pre><code>{ \"token\": string, \"expiresIn\": 3600 }</code></pre>",
    "customStyles": "h3 { margin-top: 20px; }"
  }
}

Step 2 — find the right tab

{ "name": "list_tabs", "arguments": {} }
// → [{ "id": "tab_abc123", "name": "Docs", "bookmarkCount": 2 }]

Step 3 — add a bookmark that opens the page

{
  "name": "add_bookmark",
  "arguments": {
    "tab_id": "tab_abc123",
    "title": "Auth API",
    "url": "relay-page:api-auth.relay",
    "icon": "🔐",
    "description": "Login & token endpoints"
  }
}

The bookmark card now appears in the sidebar. Clicking it opens the page viewer directly — no browser needed.

2. Bookmark roundtrip with dedup agent

Before adding a bookmark an agent should check whether the URL already exists to avoid duplicates.

// 1. List all bookmarks across all tabs
{ "name": "list_bookmarks", "arguments": {} }

// → [{ "id": "bm_x", "url": "https://github.com", "tab_id": "tab_abc123", ... }, ...]

// 2. Check if the target URL already exists
// If found → update_bookmark to refresh title/description
// If not found → add_bookmark

{ "name": "update_bookmark",
  "arguments": {
    "tab_id": "tab_abc123",
    "bookmark_id": "bm_x",
    "fields": { "description": "Updated description" }
  }
}

3. Linked multi-page doc set agent

An agent creates a set of interconnected pages — an index that links to sub-pages.

index.relay
"<h2>Project Docs</h2>
<ul>
  <li><a href=\"setup.relay\">Setup guide</a></li>
  <li><a href=\"api.relay\">API reference</a></li>
  <li><a href=\"deploy.relay\">Deployment</a></li>
</ul>"
setup.relay
"<h2>Setup</h2>
<p>Back to
  <a href=\"index.relay\">index</a>
</p>
<ol>
  <li>Clone repo</li>
  <li>npm install</li>
</ol>"

Clicking any .relay link navigates in the same panel tab with a back button. External https:// links still open in the browser.

4. Human organises tabs manually human

  1. 1
    Press Ctrl+Shift+P and run Relay: Add Tab. Name it Work.
  2. 2
    Run Relay: Add Bookmark. Choose Work, enter a title and URL. Leave the icon blank — the favicon is fetched automatically.
  3. 3
    Click any card in the Relay sidebar to open the URL in your browser.
  4. 4
    Hover a card to reveal the × remove button.

5. Human creates and opens a page human

  1. 1
    Run Relay: New Page. Enter a title — Relay creates the .relay file in relay-pages/ and names the file from the title.
  2. 2
    Open the file directly in VS Code's editor to write content, or ask an agent to populate it via update_page.
  3. 3
    Run Relay: Open Page, pick the page from the list — it opens in a full editor tab with the Relay ddd theme applied.
  4. 4
    Navigate between pages by clicking .relay links inside the viewer. Use the ← Back button to go back.

6. Iterative page editing by an agent agent

Because update_page preserves fields you don't provide, an agent can make surgical changes without rewriting the whole document.

// Only change the title — body and styles are untouched
{
  "name": "update_page",
  "arguments": {
    "filename": "api-auth.relay",
    "title": "Auth API v2"
  }
}

// Only add custom styles — title and body are untouched
{
  "name": "update_page",
  "arguments": {
    "filename": "api-auth.relay",
    "customStyles": ".endpoint { font-family: monospace; color: var(--accent2); }"
  }
}
Pattern: for multi-section documents that an agent builds incrementally — create the page with a skeleton, then call update_page once per section with the full accumulated content. Relay always replaces the entire content field, so include previous sections in each update call.

7. Per-page prototype styling agent

Custom styles apply only to the page they're defined in and can reference the full Relay theme palette.

"customStyles": "
  /* two-column layout */
  .cols { display: grid; grid-template-columns: 1fr 1fr; gap: 16px; }

  /* status badge */
  .status-ok   { background: #1a3a2a; color: #4ade80; border-radius: 4px; padding: 2px 8px; }
  .status-warn { background: #3a2a10; color: #fbbf24; border-radius: 4px; padding: 2px 8px; }

  /* accent box */
  .highlight {
    background: var(--surface);
    border: 1px solid var(--accent);
    border-radius: var(--radius);
    padding: 16px;
  }
"

Because all pages share the same VS Code theme variable mapping, styles like var(--accent) automatically adapt whenever the user changes their VS Code theme — no extra work needed.
