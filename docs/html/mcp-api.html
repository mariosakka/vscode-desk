MCP API Reference

How AI agents connect to Desk and the full tool catalogue.

Connecting an AI Agent

Desk starts a local HTTP server (default port 3333) when VS Code loads. It speaks JSON-RPC 2.0 over HTTP POST at /mcp, following the Model Context Protocol (Streamable HTTP transport). If port 3333 is already in use (e.g. a second VS Code window), Desk automatically picks the next free port — the exact URL is shown in the MCP setup notification inside VS Code.

Claude Code (~/.claude/settings.json)

{
  "mcpServers": {
    "vscode-desk": {
      "type": "http",
      "url": "http://localhost:3333/mcp"
    }
  }
}

Raw HTTP (curl)

curl -s -X POST http://localhost:3333/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": { "name": "list_projects", "arguments": {} },
    "id": 1
  }'

Initialization handshake

MCP clients send an initialize request first. Desk responds with:

{
  "protocolVersion": "2024-11-05",
  "capabilities": { "tools": {}, "resources": {} },
  "serverInfo": { "name": "vscode-desk", "version": "0.0.1" }
}

Tools

list_projects read

Returns all projects with their IDs, names, and bookmark counts.

ParameterTypeRequiredDescription
scopestringoptional"workspace" or "global" (default: "global").

Example response

[
  { "id": "proj_abc123", "name": "Work", "bookmarkCount": 4 },
  { "id": "proj_def456", "name": "Research", "bookmarkCount": 2 }
]
list_bookmarks read

Returns bookmarks. Optionally scoped to a single project.

ParameterTypeRequiredDescription
project_idstringoptionalIf provided, returns only bookmarks in that project. If omitted, returns all bookmarks with an added project_id field.
scopestringoptional"workspace" or "global" (default: "global").

Example response (single tab)

[
  {
    "id": "bm_xyz789",
    "title": "MDN Web Docs",
    "url": "https://developer.mozilla.org",
    "icon": "data:image/png;base64,...",
    "description": "Web platform reference"
  }
]
add_bookmark write

Adds a new bookmark to a project. If icon is omitted, Desk auto-fetches and caches the site's favicon.

ParameterTypeRequiredDescription
project_idstringrequiredID of the target project.
titlestringrequiredDisplay name shown on the bookmark card.
urlstringrequiredFull URL including scheme (https://…).
iconstringoptionalEmoji or base64 data URL. Omit to auto-fetch favicon.
descriptionstringoptionalShort description shown below the title.
scopestringoptional"workspace" or "global" (default: "global").

Example

{
  "project_id": "proj_abc123",
  "title": "GitHub",
  "url": "https://github.com"
  // icon omitted → favicon auto-fetched
}
remove_bookmark write

Permanently removes a bookmark from a project.

ParameterTypeRequiredDescription
project_idstringrequiredID of the project that contains the bookmark.
bookmark_idstringrequiredID of the bookmark to remove.
scopestringoptional"workspace" or "global" (default: "global").
create_project write

Creates a new empty project.

ParameterTypeRequiredDescription
namestringrequiredHuman-readable label shown in the sidebar.
scopestringoptional"workspace" or "global" (default: "global").

Example response

{ "id": "proj_new111", "name": "AI Tools", "bookmarks": [] }
remove_project write

Removes a project and all its bookmarks. This cannot be undone.

ParameterTypeRequiredDescription
project_idstringrequiredID of the project to remove.
scopestringoptional"workspace" or "global" (default: "global").
update_bookmark read/write

Partially updates a bookmark's fields. Only the fields present in fields are changed.

ParameterTypeRequiredDescription
project_idstringrequiredProject containing the bookmark.
bookmark_idstringrequiredID of the bookmark to update.
fieldsobjectrequiredAny subset of title, url, icon, description.
scopestringoptional"workspace" or "global" (default: "global").

Example — rename a bookmark

{
  "project_id": "proj_abc123",
  "bookmark_id": "bm_xyz789",
  "fields": { "title": "MDN — HTML", "url": "https://developer.mozilla.org/en-US/docs/Web/HTML" }
}

Page Tools

Pages are .desk files in <workspace>/desk-pages/. All page tools return an error if VS Code has no workspace folder open. See the Developer Guide for the .desk file format.

list_pages read

Returns all .desk files in desk-pages/.

Parameters: none

Example response

[
  { "filename": "auth-flow.desk", "title": "Auth Flow" },
  { "filename": "api-spec.desk", "title": "API Specification" }
]
create_page write

Creates a new .desk file. The file must not already exist — use update_page to modify it.

ParameterTypeRequiredDescription
filenamestringrequiredMust end in .desk, e.g. "auth-flow.desk"
titlestringrequiredShown in the page header and panel tab.
contentstringrequiredHTML body. <script> blocks and inline event handlers (onclick, etc.) work — the page viewer uses 'unsafe-inline' CSP.
customStylesstringoptionalCSS scoped to this page only. Use var(--accent), var(--text) etc. to stay on-theme.

Returns

"created auth-flow.desk"
To add a sidebar link to this page, create a bookmark with "url": "desk-page:auth-flow.desk". Clicking the card opens the viewer.
update_page read/write

Partially updates a .desk file. Only the fields you include are overwritten; everything else is preserved.

ParameterTypeRequiredDescription
filenamestringrequiredMust already exist.
titlestringoptionalNew title.
contentstringoptionalNew full body HTML (replaces the entire body).
customStylesstringoptionalNew CSS block (replaces the entire style block).

Example — rename only

{ "filename": "api-auth.desk", "title": "Auth API v2" }
delete_page write

Permanently deletes a .desk file. Cannot be undone.

ParameterTypeRequiredDescription
filenamestringrequiredMust already exist.

Resources (Agent Self-Documentation)

Desk exposes three MCP resources that describe itself. Well-behaved MCP clients read these automatically after initialize. You can also fetch them manually.

List resources

{ "jsonrpc": "2.0", "method": "resources/list", "params": {}, "id": 1 }

// response
{
  "resources": [
    { "uri": "desk://guide/quick-start",       "name": "Desk Agent Quick-Start",       "mimeType": "text/markdown" },
    { "uri": "desk://guide/desk-page-format",   "name": "Desk Page Format (.desk)",     "mimeType": "text/markdown" },
    { "uri": "desk://guide/skill-format",       "name": "Skill Format",                "mimeType": "text/markdown" }
  ]
}

Read a resource

{ "jsonrpc": "2.0", "method": "resources/read",
  "params": { "uri": "desk://guide/quick-start" }, "id": 2 }

// response
{
  "contents": [{ "uri": "desk://guide/quick-start", "mimeType": "text/markdown", "text": "# Desk Agent Quick-Start\n..." }]
}

Error Handling

On success, the JSON-RPC result is populated. On failure, an error object is returned with code -32603 and a human-readable message. HTTP status is always 200 for valid JSON-RPC calls.

{
  "jsonrpc": "2.0",
  "error": {
    "code": -32603,
    "message": "Project not found: proj_abc123"
  },
  "id": 1
}
Parse errors (-32700) and unknown routes return HTTP 400 / 404 respectively.

Favicon Auto-fetch Behaviour

When add_bookmark is called without an icon, Desk:

  1. Checks an in-memory cache keyed by hostname (TTL: 30 days).
  2. If not cached, fetches /favicon.ico from the host.
  3. If that fails, parses the homepage HTML for a <link rel="icon"> tag and fetches that URL.
  4. Stores the result as a base64 data: URL in ~/.desk/global/.
  5. Falls back to 🌐 if all attempts fail.
To skip auto-fetch and use a specific emoji or data URL, pass the icon field explicitly.
