User Guide

Everything you need to install Desk and start using it day-to-day.

Installation

Desk ships as a standard VS Code extension (.vsix file). Install it from the Extension Development Host during development, or from the VS Code Marketplace once published.

Install from a .vsix file

  1. 1
    Open VS Code and press Ctrl+Shift+P (macOS: ⌘⇧P) to open the Command Palette.
  2. 2
    Type Extensions: Install from VSIX… and press Enter.
  3. 3
    Browse to the desk-0.0.1.vsix file and confirm. VS Code will reload.
  4. 4
    Look for the Desk icon in the Activity Bar on the left edge of the window.

Run in development mode

  1. 1
    Clone the repository and run npm install && npm run compile.
  2. 2
    Open the folder in VS Code and press F5. A new Extension Development Host window opens with Desk active.

The Sidebar Panel

Click the Desk icon in the Activity Bar to open the sidebar. The panel has three areas:

AreaWhat it does
HeaderShows the Desk name and the workspace/global scope toggle. Colors follow your active VS Code theme automatically.
Tab barHorizontal tabs for each bookmark group. Click a tab to switch.
Bookmark gridCards for each bookmark in the selected project. Click a card to open the URL.

Creating a Project

Projects are named groups that hold bookmarks. You must create at least one project before you can add bookmarks.

  1. 1
    Open the Command Palette (Ctrl+Shift+P).
  2. 2
    Run Desk: Add Project.
  3. 3
    Type a name for the project (e.g. Work, Research, Tools) and press Enter.

The new project appears immediately in the sidebar. You can create as many projects as you like.

Adding a Bookmark

  1. 1
    Run Desk: Add Bookmark from the Command Palette.
  2. 2
    Select the project to add it to from the quick-pick list.
  3. 3
    Enter a title for the bookmark.
  4. 4
    Enter the URL (e.g. https://example.com).
  5. 5
    Optionally enter an emoji icon. Leave blank to have Desk auto-fetch the site's favicon.
  6. 6
    Optionally enter a description. Press Enter or Escape to skip.
Tip: When you leave the icon field blank, Desk fetches the favicon from the site's /favicon.ico or <link rel="icon"> tag and stores it locally. This happens once and is cached for 30 days.

Removing Bookmarks and Projects

Remove a bookmark

Run Desk: Remove Bookmark from the Command Palette. Pick the project, then the bookmark. The card disappears immediately.

Remove a project

Run Desk: Remove Project. Removing a project also removes all bookmarks inside it — this cannot be undone.

Note: You can also remove bookmarks directly from the sidebar by clicking the × button on a bookmark card.

Creating and Viewing Pages

Pages are .desk files stored in desk-pages/ inside the open workspace. They use a simple XML format with optional per-page CSS and render in a full VS Code editor tab.

Page format

<desk-page title="Page Title">
  <style>/* optional per-page CSS */</style>
  <!-- HTML body content here -->
</desk-page>

Create a new page

  1. 1
    Run Desk: New Page from the Command Palette.
  2. 2
    Enter a title. Desk creates the .desk file in desk-pages/ and names it from the title.
  3. 3
    Edit the file in VS Code's editor, or ask an AI agent to populate it via create_page / update_page.

Open a page

  1. 1
    Run Desk: Open Page from the Command Palette and pick a page from the list.
  2. 2
    Or add a bookmark with desk-page:filename.desk as the URL — clicking the card opens the viewer directly.
Navigation: Click any .desk link inside the viewer to navigate to another page. The ← Back button traces your history. External https:// links open in your browser.

Theming

Desk reads VS Code's active color theme automatically — backgrounds, text, borders, and accent colors all adapt to whatever theme you have installed (One Dark Pro, Catppuccin, Solarized, etc.). Change your VS Code theme and Desk updates instantly; no separate toggle needed.

Settings

SettingDefaultDescription
desk.mcpPort 3333 Port the embedded MCP HTTP server listens on. Change if 3333 conflicts with another local service. Requires a VS Code window reload to take effect.

Open VS Code Settings (Ctrl+,), search for Desk, and adjust there, or add it to your settings.json:

{
  "desk.mcpPort": 4444
}

Workspace vs. Global Scope

Desk stores data in two scopes you can switch between using the toggle in the sidebar header:

ScopeStorage locationWhen to use
workspace~/.desk/workspaces/<slug>/Projects and bookmarks tied to a specific repo or project folder.
global~/.desk/global/Projects and bookmarks available in every workspace.

All MCP tools accept an optional scope parameter ("workspace" or "global") to target either store from an AI agent.
