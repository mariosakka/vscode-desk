# Worktree–Workspace Linking Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When VS Code opens a linked git worktree, resolve it to the same Desk workspace as the main checkout, sharing bookmarks, skills, workflow config, and pages.

**Architecture:** A new pure `resolveWorktree()` function runs `git rev-parse --git-common-dir` at activation to detect linked worktrees. If detected, the extension derives its slug and data directory from the main checkout's path instead of the current folder. The `mcp-ports.json` registry is rekeyed from slug → path so two windows (main + worktree) sharing a slug each get their own port entry without overwriting each other. `PageReader` is constructed pointing at the main worktree's `desk-pages/` so all windows see one live set of pages.

**Tech Stack:** TypeScript, Node.js (`child_process.execSync`, `fs`), VS Code Extension API, Jest (mocked `child_process` + `fs`)

- [ ] **Step 1: Create `src/storage/worktreeResolver.ts`**

```typescript
import { execSync } from 'child_process';
import * as path from 'path';
import * as fs from 'fs';

export interface WorktreeInfo {
  isLinkedWorktree: boolean;
  mainWorktreePath: string | null;
}

export function resolveWorktree(workspacePath: string): WorktreeInfo {
  try {
    const result = execSync('git rev-parse --git-common-dir', {
      cwd: workspacePath,
      encoding: 'utf-8',
      timeout: 5000,
      stdio: ['pipe', 'pipe', 'pipe'],
    }).trim();

    if (!result) return { isLinkedWorktree: false, mainWorktreePath: null };

    const commonDir = path.isAbsolute(result)
      ? result
      : path.resolve(workspacePath, result);

    const isInsideWorkspace =
      commonDir.startsWith(workspacePath + path.sep) ||
      commonDir === path.join(workspacePath, '.git');

    if (isInsideWorkspace) return { isLinkedWorktree: false, mainWorktreePath: null };

    const mainWorktreePath = path.dirname(commonDir);
    if (!fs.existsSync(mainWorktreePath)) return { isLinkedWorktree: false, mainWorktreePath: null };

    return { isLinkedWorktree: true, mainWorktreePath };
  } catch {
    return { isLinkedWorktree: false, mainWorktreePath: null };
  }
}
```

import { resolveWorktree } from '../../storage/worktreeResolver';
import { execSync } from 'child_process';
import * as fs from 'fs';

jest.mock('child_process');
jest.mock('fs');

const mockExecSync = execSync as jest.Mock;
const mockExistsSync = fs.existsSync as jest.Mock;

describe('resolveWorktree', () => {
  beforeEach(() => {
    jest.resetAllMocks();
    mockExistsSync.mockReturnValue(true);
  });

  it('returns not-linked for main checkout (git-common-dir inside workspace)', () => {
    mockExecSync.mockReturnValue('/home/user/work/platform/.git\n');
    const result = resolveWorktree('/home/user/work/platform');
    expect(result.isLinkedWorktree).toBe(false);
    expect(result.mainWorktreePath).toBeNull();
  });

  it('returns linked worktree with main path when git-common-dir is outside workspace', () => {
    mockExecSync.mockReturnValue('/home/user/work/platform/.git\n');
    const result = resolveWorktree('/home/user/work/platform-feature-x');
    expect(result.isLinkedWorktree).toBe(true);
    expect(result.mainWorktreePath).toBe('/home/user/work/platform');
  });

  it('returns not-linked when git fails (not a git repo)', () => {
    mockExecSync.mockImplementation(() => { throw new Error('not a git repository'); });
    const result = resolveWorktree('/tmp/not-a-repo');
    expect(result.isLinkedWorktree).toBe(false);
    expect(result.mainWorktreePath).toBeNull();
  });

  it('returns not-linked when main worktree path does not exist on disk', () => {
    mockExecSync.mockReturnValue('/home/user/work/deleted/.git\n');
    mockExistsSync.mockReturnValue(false);
    const result = resolveWorktree('/home/user/work/feature');
    expect(result.isLinkedWorktree).toBe(false);
  });

  it('returns not-linked when git outputs an empty string', () => {
    mockExecSync.mockReturnValue('\n');
    const result = resolveWorktree('/home/user/work/platform');
    expect(result.isLinkedWorktree).toBe(false);
  });
});

- [ ] **Step 3: Replace the lookup with path-keyed longest-prefix match**

Edit `~/.desk/desk-proxy.js`. Replace the registry-lookup block with:

```js
const reg = JSON.parse(fs.readFileSync(registryPath, 'utf-8'));
const cwd = process.env.PWD || process.cwd();
const entries = Object.entries(reg)
  .filter(([p]) => cwd === p || cwd.startsWith(p + require('path').sep))
  .sort((a, b) => b[0].length - a[0].length);
if (!entries.length) {
  process.stderr.write('desk-proxy: no Desk workspace found for ' + cwd + '\n');
  process.exit(1);
}
const port = entries[0][1].port;
```

This works whether the current directory is the main worktree or a linked worktree: the longest matching prefix wins, so a worktree at `/home/user/work/platform-feature-x` matches its own entry before the main checkout entry at `/home/user/work/platform`.
